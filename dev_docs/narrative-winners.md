# Narrative Winners — Flow & Schema

The **Narrative Winners** tool identifies, for a given year, 3–4 “narrative winner” assets (mostly stocks; optionally one crypto or commodity) as they would have been perceived in **January of the following year**. Results are generated by OpenAI and stored under `data/narrative-winners/`.

## Purpose

To capture which investments were widely framed by analysts and media as the “winners” of a year—symbols of broader economic, technological, or monetary narratives—without using hindsight. Useful for historical analysis and studying how narratives and performance diverge over time.

## Flow

1. **User input**  
   User enters a year (e.g. `2017`) in the UI (`src/app/page.tsx`).

2. **API request**  
   Frontend sends `POST /api/narrative-winners` with body `{ "year": 2017 }`.

3. **Validation**  
   Route validates `year` as an integer in `[1980, 2030]` and checks for `OPENAI_API_KEY`.

4. **LLM call**  
   - **Model:** `gpt-4o`  
   - **System prompt:** Defines a “narrative winner” (widely discussed, framed as well-positioned, symbol of a broader narrative, mainstream/institutional coverage; no hindsight; exclude speculative/collapsing narratives). Asks for 3–4 items, mostly stocks, optionally one crypto/commodity if it truly fit (e.g. Bitcoin in 2017).  
   - **User prompt:** “Identify narrative winners for year X as perceived in January X+1.” Requests JSON with a `winners` array; each item has `name`, `ticker`, `type` (`"stock" | "crypto asset" | "commodity"`).  
   - **Response format:** `response_format: { type: "json_object" }`.

5. **Parse & normalize**  
   Response is parsed; if the model wraps in markdown code blocks, they are stripped. Each winner is normalized: `name` and `ticker` strings, `ticker` uppercased, `type` restricted to the three allowed values (default `"stock"`).

6. **Persist**  
   Payload is written to `data/narrative-winners/<year>.json` (e.g. `2017.json`). Directory is created if needed.

7. **Response**  
   Same payload is returned as JSON to the client; the UI displays the list and the saved file path.

## Schema

### API request

```json
{ "year": 2017 }
```

### API response / stored file

**Path:** `data/narrative-winners/<year>.json`

```ts
interface NarrativeWinnersPayload {
  year: number;           // e.g. 2017
  winners: NarrativeWinner[];
  generatedAt: string;    // ISO timestamp
}

interface NarrativeWinner {
  name: string;   // e.g. "Apple Inc."
  ticker: string; // e.g. "AAPL"
  type: "stock" | "crypto asset" | "commodity";
}
```

Example `data/narrative-winners/2000.json`:

```json
{
  "year": 2000,
  "winners": [
    { "name": "Cisco Systems Inc.", "ticker": "CSCO", "type": "stock" },
    { "name": "Nokia Corporation", "ticker": "NOK", "type": "stock" },
    ...
  ],
  "generatedAt": "2026-02-13T05:45:34.416Z"
}
```

## Environment

- **`OPENAI_API_KEY`** (required): Used by the API route for the GPT-4o call. Set in `.env.local`.

## Regenerating a year

Calling the API again for the same year overwrites the existing `data/narrative-winners/<year>.json` file with a new LLM result and a new `generatedAt` timestamp.

## Downstream use

- **Price data backfill** uses these files to derive the full set of tickers to fetch (see [price-data.md](price-data.md)).
- **Investment path** uses them to define cohorts: each start year’s narrative winners are the tickers we invest in for that cohort (see [investment-path.md](investment-path.md)).
